# Project 1: Multi-Server Configuration Management using Ansible

## Project Objective

Automate the configuration of multiple Linux servers.

Tasks:

* Create users
* Install packages
* Configure SSH
* Configure firewall
* Set timezone
* Update system
* Configure NTP
* Display custom MOTD

---

## Project Structure

```text
multi-server-config/
│
├── inventory/
│   ├── hosts
│
├── playbook.yml
│
├── roles/
│   ├── common/
│   │   ├── tasks/
│   │   │     main.yml
│   │   ├── handlers/
│   │   │     main.yml
│   │   ├── templates/
│   │   │     motd.j2
│   │   └── vars/
│   │         main.yml
│
└── ansible.cfg
```

---

## Inventory

```ini
[web]
192.168.1.10

[db]
192.168.1.11

[all:vars]
ansible_user=ubuntu
ansible_ssh_private_key_file=~/.ssh/id_rsa
```

---

## Variables

```yaml
users:
  - devops
  - developer

packages:
  - git
  - curl
  - vim
  - unzip
  - net-tools

timezone: Asia/Kolkata
```

---

## Tasks

```yaml
---

- name: Update packages
  apt:
    update_cache: yes

- name: Install packages
  apt:
    name: "{{ packages }}"
    state: present

- name: Create users
  user:
    name: "{{ item }}"
    state: present
    shell: /bin/bash
  loop: "{{ users }}"

- name: Set timezone
  timezone:
    name: "{{ timezone }}"

- name: Copy MOTD
  template:
    src: motd.j2
    dest: /etc/motd

- name: Enable UFW
  ufw:
    state: enabled

- name: Allow SSH
  ufw:
    rule: allow
    port: "22"

- name: Restart SSH
  service:
    name: ssh
    state: restarted
```

---

## Playbook

```yaml
---

- hosts: all
  become: yes

  roles:
    - common
```

---

## Resume Points

* Automated Linux server configuration using Ansible roles.
* Reduced provisioning time by 85%.
* Managed users, packages, firewall, and SSH configuration across multiple servers.
* Built reusable Ansible roles following Infrastructure as Code principles.

# Project 2: LAMP Stack Automation

## Objective

Deploy a complete LAMP stack automatically.

```
Client
      ↓
Apache
      ↓
PHP
      ↓
MySQL
```

---

## Directory

```text
lamp-deployment/

inventory/

roles/

apache/

mysql/

php/

playbook.yml
```

---

## Apache Tasks

```yaml
- name: Install Apache
  apt:
    name: apache2
    state: present

- name: Start Apache
  service:
    name: apache2
    state: started
    enabled: yes

- name: Copy Virtual Host
  template:
    src: vhost.conf.j2
    dest: /etc/apache2/sites-available/000-default.conf

- name: Restart Apache
  service:
    name: apache2
    state: restarted
```

---

## PHP Tasks

```yaml
- name: Install PHP
  apt:
    name:
      - php
      - libapache2-mod-php
      - php-mysql
    state: present
```

---

## MySQL Tasks

```yaml
- name: Install MySQL
  apt:
    name: mysql-server
    state: present

- name: Start MySQL
  service:
    name: mysql
    state: started
    enabled: yes
```

---

## Deploy Website

```yaml
- name: Copy website
  copy:
    src: index.php
    dest: /var/www/html/index.php
```

---

## Sample PHP Page

```php
<?php
phpinfo();
?>
```

---

## Playbook

```yaml
---

- hosts: web
  become: yes

  roles:
    - apache
    - php
    - mysql
```

---

## Bonus Features

* Virtual Hosts
* SSL using Let's Encrypt
* Database creation
* User creation
* Backup automation

---

## Resume Points

* Automated deployment of Apache, PHP, and MySQL using reusable Ansible roles.
* Configured web server, database, and application deployment with a single playbook.
* Reduced deployment time from approximately 30 minutes to under 5 minutes.

# Project 3: Jenkins + Ansible CI/CD Pipeline

## Architecture

```text
Developer

     │

GitHub Repository

     │

Jenkins

     │

Ansible

     │

Target Server

     │

Docker

     │

Running Application
```

---

## Objective

Whenever code is pushed to GitHub:

* Jenkins detects the change.
* Jenkins checks out the latest code.
* Jenkins triggers an Ansible playbook.
* Ansible deploys the application on the target server.

---

## Folder Structure

```text
jenkins-ansible-cicd/

Jenkinsfile

inventory/

deploy.yml

roles/

docker/

app/
```

---

## Jenkinsfile

```groovy
pipeline {
    agent any

    stages {

        stage('Clone') {
            steps {
                git 'https://github.com/yourrepo/demo-app.git'
            }
        }

        stage('Deploy') {
            steps {
                sh '''
                ansible-playbook \
                -i inventory/hosts \
                deploy.yml
                '''
            }
        }
    }
}
```

---

## Inventory

```ini
[app]
192.168.1.20

[app:vars]
ansible_user=ubuntu
```

---

## Deploy Playbook

```yaml
---

- hosts: app
  become: yes

  roles:
    - docker
    - app
```

---

## Docker Role

```yaml
- name: Install Docker
  apt:
    name: docker.io
    state: present

- name: Start Docker
  service:
    name: docker
    state: started
```

---

## Deploy Application

```yaml
- name: Copy Dockerfile
  copy:
    src: Dockerfile
    dest: /home/ubuntu/

- name: Build Image
  command:
    docker build -t demoapp .

- name: Run Container
  command:
    docker run -d -p 80:80 demoapp
```

---

## Jenkins Workflow

```text
Git Push

↓

GitHub Webhook

↓

Jenkins

↓

Clone Repository

↓

Run Tests

↓

Execute Ansible

↓

Deploy to Server

↓

Application Live
```

---

## Advanced Features

* Automatic rollback
* Health checks after deployment
* Slack or email notifications
* Blue-Green deployment
* Deployment to staging and production environments using separate inventories

---

## Resume Points

* Built an automated CI/CD pipeline using Jenkins and Ansible for containerized application deployments.
* Integrated GitHub webhooks with Jenkins to enable continuous deployment.
* Automated Docker installation, image creation, and application deployment using Ansible.
* Reduced deployment time by approximately 90% while ensuring consistent and repeatable releases.
